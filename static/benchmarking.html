Benchmarking & Quality

評価と品質

Every change clears the gate. Quality is measured in BLEU against a redacted production corpus. Performance is measured against committed baselines with tolerance-gated exit codes. Ten declarative eval suites turn things like a11y, security, and tech-debt into structured findings instead of reviewer opinions.

すべての変更はゲートを通過します。品質はリダクトされた本番コーパスに対する BLEU で測定します。性能はコミット済みベースラインとの許容差を見るエグジットコードで測定します。10 種類の宣言的な評価スイートが、アクセシビリティ・セキュリティ・技術的負債といった項目を、レビュアーの意見ではなく構造化された所見として扱います。

01 · The Three Gates
01 · 三つのゲート

Three independent checks, every change.

変更ごとに三つの独立した検査。

Each gate measures a different thing and can fail independently. The CI pre-push hook runs the full set in under two minutes. An opt-in long-run suite extends to live ASR / translation / TTS regression against saved meetings.

各ゲートは異なる対象を測定し、独立して失敗します。プッシュ前の CI フックは 2 分以内に全セットを走らせます。オプトインの長時間スイートは、保存済み会議に対するライブ ASR ／翻訳／ TTS の回帰検査まで拡張します。

2

Quality · translation, transcription, voice.

品質・翻訳、文字起こし、音声。

A 500-record redacted production corpus pins BLEU scores per direction. Today, the FP8+flashinfer baseline reads JA→EN 75.89 and EN→JA 82.30. ASR is gated on word error rate against a Japanese meeting corpus. TTS quality is gated by mean opinion score plus a concurrent-load test.

500 件のリダクト済み本番コーパスで方向別の BLEU を固定します。本番の FP8+flashinfer ベースラインは今日、JA→EN 75.89、EN→JA 82.30 です。ASR は日本語会議コーパスに対する単語誤り率で判定し、TTS 品質は平均オピニオンスコア（MOS）と同時負荷テストで判定します。

Slide translation has its own regression. The Phase 4a fixture deck translates through four directions, and the gate fails on any parse failure, ID-coverage gap, or run-count mismatch. Live-meeting translation runs a 72-pair JA↔EN burst suite that replays the exact request shape that triggered a cuBLAS crash in April 2026. That proves the hardening covered the failure mode.

スライド翻訳には独自の回帰検査があります。Phase 4a のフィクスチャ・デッキを 4 方向で翻訳し、パース失敗・ID 欠落・ラン数の不一致のいずれかでゲートが失敗します。ライブ会議翻訳には、2026 年 4 月の cuBLAS クラッシュを引き起こした正確なリクエスト形状を再実行する 72 組の JA↔EN バーストスイートがあり、補強が当該障害モードをカバーしたことを実証します。

production corpus → BLEU per direction → ASR WER → TTS MOS → slide regression → cuBLAS burst
Why a redacted corpus mattersリダクト済みコーパスの意義
Real production utterances catch failure modes that synthetic benchmarks miss: interrupted speech, mixed-script names, three-way code-switching in the same sentence. Redaction removes proper nouns and sensitive content but keeps the linguistic shape that makes the BLEU score load-bearing. 実際の本番発話は、合成ベンチマークでは捕捉できない障害モード（途切れ発話、混合表記の固有名詞、同一文での三言語コードスイッチ）を捕まえます。リダクションは固有名詞や機微な内容を除去しつつ、BLEU スコアを意味あるものにする言語的形状は保ちます。
3

Performance · three exit codes, one verdict.

性能・三つの終了コードと一つの判定。

The autosre perf harness runs two workloads against the same vLLM instance: translation (priority preempts) and coding (priority preempted). It runs them in an isolation phase and a contention phase, compares numbers against a committed JSON baseline, and exits with one of three codes. Exit code 2 is what blocks a merge.

autosre perf ハーネスは、同一の vLLM インスタンス上で翻訳（優先プリエンプト側）とコーディング（プリエンプト対象側）の 2 ワークロードを動かします。隔離フェーズと競合フェーズで実行し、コミット済み JSON ベースラインと比較し、3 つの終了コードのいずれかで終了します。マージをブロックするのはエグジットコード 2 です。

The contention phase is the one that matters for live coexistence. Translation TTFT p99 sits at 194.7 ms against a roughly 400 ms live-path SLO. Coding TPS p50 in contention is 27.29 tok/s, well above the priority-preempt floor.

ライブ共存で意味を持つのは競合フェーズです。翻訳の TTFT p99 は、約 400 ms のライブ経路 SLO に対し 194.7 ms に収まっています。競合下のコーディング TPS p50 は 27.29 tok/s で、優先プリエンプト下限を大きく上回ります。

0cleanクリーン → 1warn警告 → 2fail失敗

The current production baseline lives at repos/auto-sre/benchmarks/baselines/gb10_qwen36_fp8_flashinfer.json. Every recipe edit, container image swap, and proxy-code change has to re-clear it before it can ship.

現行の本番ベースラインは repos/auto-sre/benchmarks/baselines/gb10_qwen36_fp8_flashinfer.json にあります。レシピ編集、コンテナイメージ差し替え、プロキシ・コード変更は、いずれも出荷前にこれを再度クリアする必要があります。

4

Refinement · the second-pass latency budget.

リファインメント・二次パスのレイテンシ予算。

Refinement re-translates each finalised utterance with N prior segments as context. That costs latency but improves accuracy on pronouns and references. The sweep below shows the tradeoff against 107 finalised utterances of a real meeting. The gate that ships sits at window 4: the biggest accuracy lift without crossing the 400 ms p50 SLO.

リファインメントは、確定済みの各発話を前の N 個のセグメントを文脈として再翻訳します。代名詞や指示語の精度は上がりますが、レイテンシのコストが乗ります。下のスイープは実会議の 107 件の確定発話に対してこのトレードオフを示しています。出荷ゲートは ウィンドウ 4。p50 SLO 400 ms を超えずに最大の精度向上が得られる点です。

05 · The Eval Suites
05 · 評価スイート

Ten declarative suites, one YAML each.

10種の宣言的スイート、1つのYAMLにつき1つ。

Each suite is pure data: agent count, roles, prompts, target globs. Findings are structured (category + severity + file + reason) instead of free-text reviews. Suites live at repos/auto-sre/autosre/eval/suites/*.yaml. Adding a new one is one file.

各スイートは純粋なデータです。エージェント数、役割、プロンプト、対象 glob。所見は自由記述のレビューではなく構造化されたもの（カテゴリ＋深刻度＋ファイル＋理由）です。スイートは repos/auto-sre/autosre/eval/suites/*.yaml にあり、追加は 1 ファイル書くだけです。

a11y×3
Accessibility scan: contrast, ARIA, keyboard-only flows.アクセシビリティ：コントラスト、ARIA、キーボード操作。
coverage×3
Test-coverage gaps: untested code paths and assertion strength.テストカバレッジの欠落：未検証パスと検証強度。
dead-code×2
Unreachable code, orphan exports, never-imported modules.到達不能コード、孤立エクスポート、未インポートのモジュール。
duplication×2
Copy-pasted logic that should be extracted into a shared helper.共有ヘルパーに抽出すべきコピー貼り付けロジック。
i18n×3
Untranslated strings, hard-coded language paths, locale traps.未翻訳文字列、ハードコードされた言語経路、ロケールの罠。
leakage×3
Secrets / PII / credentials in code, configs, or logs.コード・設定・ログ中の秘匿情報・PII・認証情報。
quality×3
General code-quality readings: clarity, naming, structural fit.一般的なコード品質：明瞭性、命名、構造的適合性。
security×4
Injection vectors, auth gaps, unvalidated input, supply-chain.注入経路、認証の欠落、未検証入力、サプライチェーン。
tech-debt×2
TODOs, workarounds, deprecation timelines, refactor opportunities.TODO、暫定対応、非推奨スケジュール、リファクタリング候補。
ui-consistency×3
Design-token drift, mixed spacing/colour values, inconsistent states.デザイントークンのドリフト、間隔・色の混在、状態の不整合。
06 · Drift Watchlist
06 · ドリフト監視

Pinned models, watched for upstream change.

ピン留めされたモデルを上流の変化に対して監視。

Every model the live stack depends on is pinned by exact revision. A drift checker (scripts/bench/check_model_drift.py) walks scripts/bench/watchlist.yaml, queries Hugging Face for newer sibling releases, and probes a closed-source watchlist for the moment a previously-announced model flips to open weights.

ライブスタックが依存する全モデルは厳密なリビジョンでピン留めされています。ドリフトチェッカー（scripts/bench/check_model_drift.py）は scripts/bench/watchlist.yaml を走査し、Hugging Faceで新しい兄弟リリースを照会するほか、クローズドソースの監視対象が公開重みに切り替わる瞬間を探知します。

07 · Case Study · Phase 6
07 · ケーススタディ・Phase 6

Two candidates, two clean fails, no promotion.

二つの候補、二つの明確な失敗、昇格なし。

In April 2026 we ran a structured campaign to evaluate two upgrade paths off the FP8+flashinfer baseline. Speculative decoding via DFlash, and a native NVFP4 quantisation. Neither cleared the three-tier gate. The retrospective at reports/phase6/decision_gate_2026-04-25.md captures the root causes and the exact upstream-bug triggers that would re-open each path. This is what the gate process produces when it works.

2026 年 4 月、FP8+flashinfer ベースラインから外れる 2 つのアップグレード経路を評価する構造化キャンペーンを実施しました。DFlash による投機的デコーディングと、ネイティブ NVFP4 量子化です。いずれも三段階ゲートを通過できませんでした。レトロスペクティブ reports/phase6/decision_gate_2026-04-25.md に根本原因と、各経路を再オープンする上流バグの正確なトリガーが記録されています。これがゲート・プロセスがうまく機能したときの姿です。

When to retry再試行のタイミング
Each failed candidate has a concrete upstream trigger that would unblock a retry. DFlash needs a vLLM patch that lets flashinfer cover non-causal drafter attention. That is unlikely soon, since block diffusion is inherently non-causal. NVFP4 needs an SM121 native FP4 kernel fix in CUTLASS / vLLM that makes flashinfer_mm_fp4 stop returning Error Internal. The recipes are committed and ready to retry the moment those land. 失敗した各候補には、再試行を解禁する具体的な上流トリガーがあります。DFlash は、flashinfer が非因果ドラフター・アテンションをカバーできるようにする vLLM パッチが必要です。ブロック拡散は本質的に非因果なので、近い将来は難しいでしょう。NVFP4 は、flashinfer_mm_fp4 が Error Internal を返さなくなるよう、CUTLASS ／ vLLM での SM121 ネイティブ FP4 カーネル修正が必要です。レシピはコミット済みで、修正が降りた瞬間に再試行できます。

08 · Reference08 · 参考

Terminology用語集

Every acronym used above, in one place. The dotted underlines in the body link here. 本文で使われている略語を一箇所にまとめています。本文中の点線下線はこのセクションへのリンクです。

BLEU
Bilingual Evaluation Understudy. A precision score for translations against one or more reference translations. Higher is better.機械翻訳の自動評価指標。1つ以上の参照翻訳に対する精度スコア。値が高いほど良い。
WER
Word Error Rate. The standard ASR accuracy metric: substitutions + deletions + insertions divided by reference length. Lower is better.単語誤り率。ASR 精度の標準指標。置換・削除・挿入の合計を参照長で割る。値が低いほど良い。
MOS
Mean Opinion Score. Subjective quality rating for synthesised speech, usually on a 1–5 scale.平均オピニオンスコア。合成音声の主観品質評定、通常1〜5段階。
TTFT
Time To First Token. Milliseconds from request arrival to the first output token. The latency a user perceives before anything shows up.最初のトークンが返るまでの時間（ms）。ユーザーが「何か返ってきた」と感じるまでのレイテンシ。
TPS
Tokens Per Second. The steady-state generation throughput once TTFT has elapsed. Reported per-request (p50, p95) and aggregate.毎秒トークン数。TTFT 後の定常生成スループット。リクエスト単位（p50、p95）と集約で計測。
p50 / p95 / p99
Percentiles of a metric distribution. p99 is the slow tail, the metric we care about for SLO compliance.指標分布のパーセンタイル。p99 はテイル。SLO 遵守で最も重要な数値。
Baseline
A committed JSON snapshot of expected performance under a fixed configuration. New runs compare to this; tolerances decide pass/warn/fail.固定構成下での期待性能のコミット済みJSONスナップショット。新規実行はこれと比較し、許容差で合格／警告／失敗を判定。
Contention
A measurement phase where the translation and coding workloads run concurrently against the same vLLM instance. The realistic live-coexistence scenario.翻訳とコーディングのワークロードを同一 vLLM インスタンス上で同時実行する計測フェーズ。現実的なライブ共存シナリオ。
Isolation
A measurement phase where a single workload runs alone. The "best case" reference against which contention numbers are compared.単一ワークロードのみを実行する計測フェーズ。競合数値と比較される「最良ケース」のリファレンス。
Priority preempt
vLLM scheduling that lets a higher-priority request evict a running lower-priority one. Translation (priority −10) preempts coding (priority +10) when slots saturate.高優先度リクエストが実行中の低優先度リクエストを退避させるvLLMスケジューリング。スロット飽和時、翻訳（優先度−10）がコーディング（優先度+10）をプリエンプトする。
Gate
A merge-blocking check. Three gates here: quality (BLEU / WER / MOS), performance (TTFT / TPS vs baseline), refinement (on-p99 latency).マージをブロックする検査。ここでは3つ：品質（BLEU / WER / MOS）、性能（TTFT / TPSのベースライン比較）、リファインメント（on-p99レイテンシ）。
Drift
When committed artifacts (CSS, baselines, model pins) diverge from what their sources would now produce. The CI gate trips on drift.コミット済みアーティファクト（CSS、ベースライン、モデルピン）が、ソースが今生成するものと乖離した状態。CIゲートはドリフトで失敗する。
